# Инструкции для агентов

Эти инструкции распространяются на весь репозиторий.

## Контекст проекта

Universal WebSocket Platform — open-source платформа для создания, настройки, развертывания и эксплуатации независимых WebSocket-серверов без написания инфраструктурного кода.

Перед внесением изменений прочитайте:

1. `.ai/PROJECT_CONTEXT.md`
2. `spec/README.md`
3. `spec/current-state.md`
4. `spec/decisions.md`
5. Все спецификации, относящиеся к задаче

## Правила работы

- Считайте спецификации источником истины в вопросах предполагаемого поведения и архитектуры.
- Не выходите за рамки текущей вехи и явно обозначенного объема задачи.
- Не добавляйте код приложения во время M0 Bootstrap.
- Отдавайте предпочтение небольшим изменениям, которые легко проверить и для которых ясно обоснование.
- Сохраняйте Technology Neutrality, пока выбор технологии не будет закреплен принятым решением.
- Фиксируйте значимые архитектурные решения в виде ADR в `spec/adr/`.
- Обновляйте `spec/current-state.md`, когда возможности репозитория существенно меняются.
- Обновляйте документацию вместе с изменением, которое она описывает.
- Никогда не добавляйте в репозиторий секреты, учетные данные, сгенерированные артефакты и локальные файлы окружения.
- Не создавайте commit без явного указания.

## Проверка

При изменении только документации проверяйте ссылки, заголовки, терминологию и итоговую структуру репозитория. После начала реализации используйте команды проекта, которые к тому моменту будут описаны в репозитории.
