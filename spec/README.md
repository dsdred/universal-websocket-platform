# Содержание спецификаций

Этот каталог служит источником истины для замысла продукта, архитектурных принципов, текущего состояния и зафиксированных решений Universal WebSocket Platform.

## Содержимое

- [`00-product/vision.md`](00-product/vision.md) — миссия, целевая аудитория и направление развития продукта.
- [`01-principles/architecture-principles.md`](01-principles/architecture-principles.md) — принципы, которыми следует руководствоваться при принятии проектных решений.
- [`adr/`](adr/) — записи об архитектурных решениях.
- [`current-state.md`](current-state.md) — фактическое описание текущего состояния репозитория.
- [`decisions.md`](decisions.md) — перечень принятых и ожидающих принятия решений.

## Правила оформления спецификаций

- Описывайте замысел отдельно от реализованного состояния.
- Используйте нормативные формулировки осознанно: **должен** — для обязательных требований, **следует** — для настоятельных рекомендаций, **может** — для необязательного поведения.
- Оформляйте каждое архитектурное решение отдельным ADR, если оно создает долгосрочное ограничение или предполагает существенный компромисс.
- Используйте относительные ссылки между спецификациями, чтобы они работали локально и на площадках размещения репозитория.
- Обновляйте `current-state.md` при смене вехи или фактическом появлении новой возможности.
