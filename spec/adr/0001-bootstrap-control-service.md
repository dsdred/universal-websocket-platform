# ADR 0001: Базовая реализация Control Service

## Статус

Принято.

## Контекст

После этапа документационного Bootstrap проекту требуется первый исполняемый компонент: минимальный HTTP Control Service с endpoint проверки состояния.

## Решение

- Использовать Go 1.25.
- Использовать Chi Router для маршрутизации HTTP-запросов.
- Разместить точку входа в `cmd/control-service`.
- Разместить внутренние пакеты HTTP Server, Configuration и логирования в `internal/`.
- Использовать `slog` для логирования.
- Создавать зависимости явно, без сторонних DI-библиотек и глобальных переменных.

## Последствия

- Команда `go run ./cmd/control-service` запускает HTTP Server на порту `8080` по умолчанию.
- Порт можно задать через переменную окружения `PORT`.
- Endpoint `GET /health` возвращает HTTP 200 и `{"status":"ok"}`.
- Go и Chi становятся принятыми технологиями для текущего Control Service.
