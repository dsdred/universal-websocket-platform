# Universal WebSocket Platform v0.1.0-alpha

The first alpha release establishes a working Control Service foundation and the basic domain model for managing Workspace, Configuration, and ConfigurationVersion.

## Highlights

- Executable Go Control Service with startup Configuration, structured logging, and graceful shutdown.
- In-memory APIs for managing Workspace and Configuration.
- Basic ConfigurationVersion lifecycle: Draft creation, publishing, and archiving.

## Included

- `GET /health` endpoint.
- CRUD API for Workspace.
- Nested CRUD API for Configuration.
- ConfigurationVersion creation with automatic numbering.
- ConfigurationVersion publishing with automatic archiving of the previous Published Version.
- Manual ConfigurationVersion archiving.
- Unit and HTTP tests for implemented components.

The complete change list is available in [`CHANGELOG.md`](../../../CHANGELOG.md).

## Known limitations

- All domain data is stored only in process memory.
- PostgreSQL is not connected.
- Runtime and WebSocket listener are not implemented.
- Admin UI is not implemented.
- Validation, Rollback, and Snapshot are not implemented.
- The race detector was not run in the current Windows environment without CGO.
