# Universal WebSocket Platform v0.1.0-alpha

Первый alpha-релиз фиксирует рабочий фундамент Control Service и базовую доменную модель управления Workspace, Configuration и ConfigurationVersion.

## Основные возможности

- Исполняемый Control Service на Go с Configuration запуска, структурированным логированием и graceful shutdown.
- In-memory API для управления Workspace и Configuration.
- Базовый жизненный цикл ConfigurationVersion: создание Draft, публикация и архивирование.

## Состав релиза

- Endpoint `GET /health`.
- CRUD API для Workspace.
- Вложенный CRUD API для Configuration.
- Создание ConfigurationVersion с автоматической нумерацией.
- Публикация ConfigurationVersion с автоматическим архивированием предыдущей Published Version.
- Ручное архивирование ConfigurationVersion.
- Unit- и HTTP-тесты реализованных компонентов.

Полный список изменений приведен в [`CHANGELOG.md`](../../../CHANGELOG.md).

## Известные ограничения

- Все доменные данные хранятся только в памяти процесса.
- PostgreSQL не подключен.
- Runtime и WebSocket listener не реализованы.
- Admin UI не реализован.
- Validation, Rollback и Snapshot не реализованы.
- Race detector не запускался в текущей Windows-среде без CGO.
