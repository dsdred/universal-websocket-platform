# Участие в разработке

Благодарим за участие в развитии Universal WebSocket Platform.

## Перед началом работы

1. Прочитайте `README.md` и `spec/README.md`.
2. Проверьте реализованные возможности в `spec/current-state.md`.
3. Ознакомьтесь с `spec/decisions.md` и относящимися к задаче ADR.
4. Убедитесь, что предлагаемое изменение не выходит за рамки текущей вехи.

## Внесение изменений

- Для существенных изменений продукта или архитектуры создавайте отдельный issue или предложение с четко определенной темой.
- Делайте pull request достаточно небольшим, чтобы его можно было проверить как одно целостное изменение.
- Опишите проблему, выбранный подход и связанные с ним компромиссы.
- Обновляйте спецификации при изменении поведения, ограничений или состояния проекта.
- Добавляйте ADR для решений, которые накладывают долгосрочные архитектурные ограничения.
- Не смешивайте с функциональным изменением не связанное с ним форматирование или рефакторинг.
- Не добавляйте секреты, учетные данные и настройки, зависящие от конкретного компьютера.

## Ограничения M0 Bootstrap

M0 включает только документацию. В рамках этой вехи не добавляйте код приложения, Go module, каталоги сервисов, инфраструктуру развертывания и хранения данных или компоненты Runtime.

## Стиль документации

- Используйте лаконичный Markdown и содержательные заголовки.
- Отделяйте текущее поведение от планов на будущее.
- Для файлов репозитория используйте относительные ссылки.
- Всегда используйте название проекта **Universal WebSocket Platform** без изменений.

## Проверка

Перед отправкой изменения:

- Проверьте измененные файлы на ясность и внутреннюю согласованность.
- Проверьте относительные ссылки и структуру репозитория.
- Убедитесь, что `spec/current-state.md` остается актуальным.
- Выполните все описанные проверки, применимые к изменению.

## Commit и pull request

Используйте повелительную форму и указывайте область изменения в заголовке commit, например:

```text
docs: define bootstrap specification structure
```

В pull request кратко опишите изменение, перечислите выполненные проверки и укажите принятые решения или дальнейшие задачи.
